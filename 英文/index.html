    # why need a IT developer

    Investment banks need IT developers for several reasons:

    1. Trading and risk management systems: Investment banks rely heavily on complex trading and risk management systems to execute trades and manage their risk exposure. IT developers are responsible for designing, developing, and maintaining these systems, which require expertise in areas such as software engineering, database management, and system integration.
    2. Data analysis and modeling: Investment banks generate and process large amounts of data, which requires advanced data analytics and modeling capabilities. IT developers can help design and implement data analytics platforms, which allow investment banks to analyze large data sets, identify trends, and make better-informed investment decisions.
    3. Regulatory compliance: Investment banks are subject to a wide range of regulatory requirements, which require comprehensive reporting and compliance systems. IT developers can help design and implement these systems, which need to be robust, secure, and able to generate complex reports on demand.
    4. Client services and operations: Investment banks provide a wide range of services to clients, including corporate finance, M&A, and asset management. IT developers can help design and implement client-facing applications and systems, which improve customer experience and streamline operations.

    Overall, IT developers are critical to the success of investment banks, as they help design, develop, and maintain the complex systems and applications that are essential to the bank’s operations, risk management, and regulatory compliance.

    # pointer

    In computer programming, a pointer is a variable that stores the memory address of another variable or object. Pointers provide a way to manipulate memory directly and efficiently, without the need to copy or move large amounts of data.

    A pointer variable contains the memory address of another variable, rather than the actual value of the variable itself. By using pointers, a program can access and modify the value of a variable directly in memory, instead of working with copies of the variable’s value.

In other words, polymorphism enables a single interface to be used to represent multiple different types of objects.

    # cluster indexes and non cluster indexes

    Clustered Indexes:

    • A clustered index determines the physical order of data in a table.
    • A table can have only one clustered index.
    • A clustered index is created using a primary key or a unique key constraint. If neither of these exist, the database engine will create a clustered index using a system-generated unique key.
    • When a clustered index is created, the data in the table is physically reorganized to match the order of the index key, which can improve query performance for range-based queries.
    • Because the data is stored in the order of the clustered index, it is faster to retrieve data from the table using the clustered index than using a non-clustered index or a table scan. Non-Clustered Indexes:
    • A non-clustered index is a separate structure from the table that stores the index key and a pointer to the location of the data in the table.
    • A table can have multiple non-clustered indexes.
    • A non-clustered index is created using a CREATE INDEX statement.
    • When a non-clustered index is created, the index key is sorted in the index structure, and a pointer to the location of the data in the table is stored with each index key.
    • Because the data is not physically reorganized in a non-clustered index, it can take longer to retrieve data from the table using a non-clustered index than using a clustered index or a table scan.

    # difference between processes and threads

    In computing, a process is an instance of a program being executed by a computer. It consists of a set of instructions and data that are loaded into memory and managed by the operating system. A process has its own memory space, system resources, and can run independently of other processes.

    A thread, on the other hand, is a lightweight process that can run in parallel with other threads within the same process. Threads share the same memory space and system resources as the process they belong to. Each thread has its own stack and registers, but shares the heap memory and other resources with the other threads in the process.

    # asynchronous and synchronous

    同步就是整个处理过程顺序执行，当各个过程都执行完毕，并返回结果。是一种线性执行的方式，执行的流程不能跨越。一般用于流程性比较强的程序，比如用户登录，需要对用户验证完成后才能登录系统。 Synchronization means that the entire processing process is executed sequentially, and when each process is executed, the result is returned. It is a linear execution method, and the execution process cannot be crossed. It is generally used for programs with strong procedures, such as user login, which needs to be authenticated before logging into the system.

    异步则是只是发送了调用的指令，调用者无需等待被调用的方法完全执行完毕；而是继续执行下面的流程。是一种并行处理的方式，不必等待一个程序执行完，可以执行其它的任务，比如页面数据加载过程，不需要等所有数据获取后再显示页面。 Asynchronous means that the call instruction is only sent, and the caller does not need to wait for the called method to be completely executed; instead, it continues to execute the following process. It is a way of parallel processing. You don’t have to wait for a program to finish executing, you can perform other tasks, such as the page data loading process, and you don’t need to wait for all the data to be obtained before displaying the page.

    # multitasking vs multithreading

    multitasking the ability of an operating system to run multiple tasks or processes concurrently on a single processor or multiple processors

    multithreading is a technique used within a single process to allow multiple threads of execution to run concurrently

    In summary, multitasking is the ability to run multiple processes or tasks concurrently, while multithreading is the ability of a single process to execute multiple threads of execution concurrently. Multitasking can be achieved through multiprocessing or multithreading, while multithreading is a specific technique used within a single process.

    # connection in/out

    A “connection in” refers to data being transferred from an external source or device into a local device or network. For example, when a user accesses a website from their computer, the data is being transferred from the website’s server (external source) into the user’s computer (local device), so this is a “connection in”. A “connection out” refers to data being transferred from a local device or network to an external device or network. For example, when a user sends an email from their computer to another user’s email address, the data is being transferred from the user’s computer (local device) to the email server (external device), so this is a “connection out”.

    # DDOS

    DDoS (Distributed Denial of Service) is a type of cyber attack that is designed to disrupt normal traffic to a target server or network by overwhelming it with a flood of traffic from multiple sources. In a DDoS attack, a large number of compromised devices, called botnets or zombies, are used to send traffic to the target, making it difficult or impossible for legitimate users to access the network or server.

    # linkedlist

    A linked list is a data structure used to store a collection of elements. It consists of a sequence of nodes, where each node contains a value and a reference (or “pointer”) to the next node in the sequence. The first node is called the “head” of the linked list, and the last node is called the “tail.”

    # stack overflow

    Stack overflow is a situation that can occur in computer programs when the call stack, which is a data structure used by the program to keep track of function calls, becomes full and has no more space to add new function calls.

    If a program recursively calls a function too many times, or if a function calls other functions that use a lot of stack space, the call stack can become full and cause a stack overflow.

    require restructuring the code or using other techniques to reduce the amount of stack space used by the program.

    # tree preorder

    Preorder traversal and postorder traversal are two methods used to traverse or visit each node in a binary tree. In a binary tree, each node has at most two child nodes, referred to as the left child and the right child. The order in which the nodes are visited can affect the outcome of some algorithms that operate on the tree. Preorder traversal is a depth-first traversal method that visits the nodes in the following order:

    1. Visit the current node.
    2. Traverse the left subtree in preorder.
    3. Traverse the right subtree in preorder.
